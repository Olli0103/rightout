# RightOut release audit - 2026-07-11

Baseline: tag `v0.1.1`, commit `87fe83b4e18b7ea5e5ddca30638016e0e99e5f97`.

## Evidence scope

Observed and reviewed: every repository file, v0.1.0..v0.1.1 diff, git tags and commits, GitHub repository metadata, branches, releases, issues, pull requests, installer behavior, public CLI, tests, catalog, and generated dummy reports.

The exact independent v0.1.0 audit text was not present in the repository, GitHub issues, pull requests, release assets, or supplied attachment directory. The v0.1.0 release body evidences only four broad blocker classes: approval boundary, filesystem, installer, and catalog provenance. Exact original finding IDs and wording are `needs_evidence`; this audit does not invent them.

Baseline tests:

- `make test`: passed 18 named tests.
- `python3 -m unittest discover -v`: failed with zero tests discovered.
- `make scan-only-dummy`: passed but described California DROP as a found exposure in a synthetic scan.
- `make e2e-dummy`: passed but did not exercise submitted/awaiting/confirmed/reappeared removal states.

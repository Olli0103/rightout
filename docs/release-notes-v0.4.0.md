# RightOut v0.4.0

RightOut 0.4.0 closes the minimum Hermes Unbroker workflow gap while retaining per-action native OpenClaw approval.

Highlights:

- 22 clean-room people-search entries, 21 bounded Brave discovery lanes, aliases/prior locations/addresses/emails/phones, and official ownership clusters;
- durable PII-safe cases, deterministic next-action queue, status, due rechecks, reappearance handling, and OpenClaw Cron-compatible scheduling surface;
- catalog-locked BeenVerified email removal with restart-safe 24-hour dedupe;
- sandbox-browser PeopleConnect form initiation with closed ARIA recipe and fail-closed CAPTCHA/ID/ambiguity handling;
- read-only pinned IMAP verification plus separate opaque confirmation-link approval;
- AES-256-GCM candidate URL tokens and separately approved direct rechecks for supported lanes;
- `confirmed_removed` only after prior removal and direct absence across the known encrypted listing set; new/unindexed listings remain a coverage gap;
- ten manifest/runtime tools, expanded security audit, installer/runtime verification, parity matrix, full production SBOM, shrinkwrap, and adversarial tests.

Release validation uses only `.invalid` profiles, mocked network/provider behavior, and synthetic state. No live broker scan, form, email, link open, or provider write was performed.

This release claims minimum workflow feature parity with Hermes Unbroker, not equivalent broker automation breadth, effectiveness, private-database coverage, dashboard/UI, family administration, or managed specialist service. Unbroker's official reference remains broader in executable broker lanes.
